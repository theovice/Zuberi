## Critical Context: The Ccode Auth Decision

**Read ZUBERI-CCODE-ASSESSMENT.md in the project** — it has the full compliance analysis, cost comparison, and architecture.

**The short version:**
- Anthropic explicitly prohibits using consumer plan OAuth tokens (Free/Pro/Max) in any product other than Claude Code and claude.ai
- OpenClaw routing requests through Max credentials = non-compliant
- Even ACP (OpenClaw's Agent Client Protocol with acpx plugin) is in a policy gray zone with consumer credentials
- **API key via platform.claude.com is the only compliant path**
- James is on the Max plan at $100/month. API billing at Zuberi's projected volumes (100K–500K tokens/month) costs $1–6/month. Break-even with Max is 8.3M tokens/month on Sonnet — far beyond any projected usage.
- $1.53 prepaid credit already on account

**Sources:** GPT deep research ("Using Claude Code with an OpenClaw Personal Assistant" and "Cost Comparison" — both PDFs in Claude project), Anthropic Claude Code Legal and Compliance docs, OpenClaw Anthropic provider docs.

### 3. Dispatch Mechanism Decision
Zuberi can't SSH to CEG from inside OpenClaw's Docker container (SSH key not mounted). Two options:

**Option A: Mount SSH key** — Add `~/.ssh:/home/node/.ssh:ro` to docker-compose.yml. Simple but exposes key to container.

**Option B: HTTP dispatch wrapper on CEG:3003 (recommended)** — Small service accepting POST with task prompt, runs ccode locally, returns JSON. Zuberi dispatches via curl. Same pattern as SearXNG, CXDB, n8n, usage tracker.

**Option C: ACP (acpx plugin)** — OpenClaw's native agent delegation. Cleanest integration but needs maturity investigation on v2026.2.26. Still requires API key auth.

## Ccode Prompt Conventions

**The pattern that works:** Detailed ccode prompts with numbered tasks, explicit file paths, verification steps, and fallback instructions.

**The pattern that fails:** Vague or open-ended prompts that require ccode to make architectural decisions.

**Before starting work:**
- Kill any existing `pnpm tauri dev` process before making changes
- Run `pnpm test` — all 13 Vitest smoke tests must pass before touching code
- Run `git status` and `git log --oneline -5` to understand current state
- Read the relevant source files before editing them

**Prompt structure:**
- Deliver prompts as chat code blocks, not .md files, unless the prompt needs to be saved and reused across sessions
- Include a Context section at the top explaining what the task is and why
- Include a "CRITICAL: Before doing anything" section with pre-flight checks
- Include numbered tasks with explicit file paths
- Include a "Do NOT" section listing things to avoid
- Include a "After implementation" section with verification steps
- Include fallback instructions if something fails

**After completing work:**
- Run `pnpm test` again — all 13 tests must still pass
- Run `pnpm tauri dev` and visually verify the changes work
- Clean up any temp files created during the work
- Report a completion summary

**Report format:**
- End with a completion summary table showing task, status, and notes
- Flag anything blocked on James separately
- Note any test failures and whether they were fixed

**General rules:**
- No jq anywhere — OpenClaw container doesn't have it
- No new npm dependencies unless absolutely necessary
- Don't modify tauri.conf.json or package.json unless the task requires it
- Don't touch Rust code unless the task requires it
- API keys NEVER in workspace files, skill files, or version-controlled code

## Lessons / Warnings

1. **No jq anywhere** — OpenClaw container doesn't have it. Skill files and TOOLS.md must use grep/sed/awk for JSON parsing.
2. **Kill pnpm tauri dev** before any ZuberiChat ccode work. Include this instruction in every ccode prompt.
3. **13 Vitest smoke tests** must pass before and after any ZuberiChat changes.
4. **API key NEVER in workspace files** — store only in CEG's ~/.bashrc or ccode config. Never in skills, TOOLS.md, or version-controlled code.
5. **Max plan = $100/month** not Pro $20/month. Cost comparisons in ZUBERI-CCODE-ASSESSMENT.md reflect this.
6. **OpenClaw container can curl to CEG** (host network via Tailscale gateway) but **cannot SSH** (no key mounted). This is the dispatch gap.
