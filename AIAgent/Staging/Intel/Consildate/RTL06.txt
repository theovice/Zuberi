### Ccode Auth Decision (RTL-012) — OPEN

**Option A: Pro subscription credentials (free)**
- Authenticate ccode on CEG via `claude login` (SSH tunnel or credential copy from KILO)
- Dispatches count against Pro weekly limit (currently 48% used)
- No extra cost, but heavy Zuberi usage could rate-limit James's own claude.ai sessions
- Critical: must NOT set ANTHROPIC_API_KEY env var on CEG, or it silently switches to pay-per-token
- ~$150 worth of API-equivalent usage included in $20/month Pro plan

**Option B: API key (pay-per-token)**
- Set ANTHROPIC_API_KEY on CEG
- Dispatches billed separately at ~$3/M input, $15/M output (Sonnet rates)
- Pro limits untouched for James's personal use
- Estimated cost: $0.05–0.15 per dispatch, <$20/month for moderate use
- $1.53 prepaid credit balance already on account

**James is currently researching the optimal path with other agents.**

## Architecture Gap: Zuberi → CEG Dispatch

**Problem identified (Session 9c):** Zuberi runs inside OpenClaw's Docker container. The container has host network access at gateway level (how curl to CEG services works). But SSH to CEG requires the private key at `C:\Users\PLUTO\.ssh\id_ed25519`, which is **not mounted** into the container.

**Two solutions:**

1. **Mount SSH key** — Add volume mount to docker-compose.yml: `~/.ssh:/home/node/.ssh:ro`. Simple but exposes the key to the container.

2. **HTTP dispatch wrapper on CEG** — Small HTTP service (like the usage tracker) that accepts task prompts via POST and runs ccode locally. Zuberi hits it with curl, same as every other CEG service. No SSH needed. Cleaner separation.

**Decision needed from James.** Recommendation: Option 2 (HTTP wrapper) — follows the established curl-based pattern for all CEG services and doesn't require mounting credentials into containers.
